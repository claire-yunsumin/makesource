# 수동 QA 체크리스트 (T7.5)

**실행 환경**: Apple Silicon Mac 필수 (이미지 생성이 MPS 전제라 인텔 기기에서는 실행 불가 — CLAUDE.md/README 참고). v0.5 내부 베타 배포 전 이 체크리스트를 전부 통과해야 한다.

사용법: 항목별로 실행하고 결과(✅/❌ + 메모)를 이 문서에 직접 기록한다. 실패 항목은 GitHub 이슈로 남기고 여기 링크를 건다.

---

## 0. 사전 준비

- [ ] 테스트용 기기의 실제 메모리 확인 (`sysctl hw.memsize`) — 16GB 이상 1대, 가능하면 8GB 1대(§8 폴백 시나리오용)
- [ ] 깨끗한 상태로 시작: `~/Library/Application Support/LocalBrush/` 삭제 (기존 설치 흔적 제거) — **주의: 실제 생성 데이터가 있다면 먼저 백업**
- [ ] [Releases](https://github.com/claire-yunsumin/makesource/releases)에서 최신 dmg 설치 (또는 로컬 빌드: `pnpm tauri build`)

## 1. M1 — 엔진 연동 (T7.0 부트스트랩 UI 포함, 최우선 리스크)

- [ ] **T1.1/T7.0 최초 설치**: dmg로 처음 실행 → 부트스트랩 풀스크린 화면 표시 → 프로파일 선택(RAM 16GB+ 기기에서 "표준" 추천 배지 확인) → 설치 시작 → 4단계(환경 준비/엔진 설치/모델 다운로드/준비 완료) 진행률이 순서대로 올라감 → 완료 시 자동으로 생성 화면 진입
- [ ] **중단 후 재개**: 모델 다운로드 중 앱 강제 종료(⌘Q 또는 kill) → 재실행 → "이어서 설치하기" 표시 → 완료된 단계 재다운로드 없이 이어서 진행
- [ ] **실패 시 UX**: 네트워크를 끄고 설치 시도 → 에러 요약 표시 → [로그 열기] 클릭 시 `logs/bootstrap.log`가 기본 앱(텍스트 편집기 등)으로 열림 → 네트워크 복구 후 [재시도] 클릭 시 이어서 진행
- [ ] **T1.2 엔진 프로세스**: 앱 실행 중 활동 모니터에서 ComfyUI(python) 프로세스 kill → 앱이 자동 재시작하는지, `logs/engine.log`에 복구 로그가 남는지 확인
- [ ] **T1.4/T1.3 실제 생성**: 생성 화면에서 프리셋 1개 + 키워드 입력 → 1024px 이미지가 정상 생성·저장되는지, 생성 중 [취소]가 실제로 중단시키는지

## 2. M2 — 생성 화면 (통합 AC)

- [ ] **핵심 플로우**: "동화같은" 스타일 + "통나무집" 키워드 입력 → 4장 생성 → 결과 중 1장 다운로드까지 **마우스 5클릭 이내** (타입 카드 선택 1 + 키워드 입력 후 생성 버튼 1 + 다운로드 버튼 1 = 3클릭이 이상적, 5클릭 초과 시 실패로 기록)
- [ ] ⌘↵로 생성, Esc로 취소가 키보드만으로 동작
- [ ] 한글 키워드 입력 시 고급 패널에서 영문 변환 미리보기가 표시됨 (용어 사전 히트/Argos 변환/원문 폴백 3가지 케이스 각각 한 번씩 시도)
- [ ] 결과 이미지 ♥ 토글, PNG 다운로드(다운로드 폴더 확인), 시드 고정 재생성이 모두 동작
- [ ] 투명 배경 옵션 체크 후 생성 → 다운로드한 PNG가 실제로 알파 채널을 가짐 (미리보기 앱에서 배경 격자 확인)

## 3. M3 — 갤러리 (통합 AC)

- [ ] 40장 이상 생성한 상태에서 갤러리 무한 스크롤이 끊김 없이 로드됨
- [ ] 키워드 검색, ♥ 필터가 정확한 결과만 보여줌
- [ ] 상세 모달: 메타 정보 표시·복사, "같은 설정으로 재생성", PNG/JPG/WebP 3포맷 내보내기 모두 동작
- [ ] 갤러리 이미지를 Finder(또는 다른 앱)로 직접 드래그 아웃 — 파일이 실제로 복사/이동됨

## 4. M4 — 에센스 스타일 (수동 평가 필요)

- [ ] 브랜드 참조 이미지 5장으로 에센스 마법사 실행 → **10분 이내** 스타일 저장 완료 (첫 실행은 Florence-2/WD14 모델 다운로드 포함 — 이 시간은 별도 기록하고 재시도 시 기준으로 10분 측정)
- [ ] 저장한 에센스 스타일로 생성 실행 → 결과 이미지가 참조 이미지의 톤(색감·분위기)을 반영하는지 **사람이 직접 평가**하고 아래에 기록:
  - 평가 메모: `______________________________________`
  - 프리셋 성공 기준 메모([05_구현_백로그.md](05_구현_백로그.md) 방침대로 프리셋 편집기의 "성공 기준 메모"에도 옮겨 자산화)

## 5. M5 — 프리셋 자산화

- [ ] 프리셋 편집 → 버전 저장 → 히스토리에서 이전 버전 복원 → 값이 정확히 되돌아옴
- [ ] A/B 비교: 같은 키워드·시드로 프리셋 2개 나란히 생성 → 좌우 결과가 시드 고정에 맞게 재현 가능한 차이만 보임
- [ ] 내보내기 → 다른 기기(또는 폴더 초기화 후)에서 가져오기 → 프리셋이 병합되어 정상 로드됨. schemaVersion이 다른 파일을 가져오면 아무 것도 바뀌지 않고 실패 메시지만 나오는지도 확인

## 6. M6 — LoRA 학습 (수동 평가 필요, 소요 시간 김)

- [ ] 이미지 30장으로 학습 마법사 실행 → 데이터셋 빌드(캡션 자동 생성) → fast 프로파일로 학습 시작
- [ ] 학습 대시보드에서 진행률·ETA·epoch 샘플 스트립이 갱신됨, [취소]가 실제로 프로세스를 중단시킴
- [ ] 학습 완주 → 스타일 목록에 자동 등록됨 → 해당 LoRA 스타일로 생성 시도 → 성공적으로 이미지가 나오는지
- [ ] 소요 시간 기록: `______` (PRD 기준 8시간 이내가 목표)

## 7. M7 — 마감 (이번 세션에서 구현한 항목의 실기기 확인)

- [ ] **T7.1 설정**: 모델 목록·용량이 Finder에서 본 `models/` 실제 용량과 일치, 캐시 정리 후 에센스 분석 재실행 시 모델이 정상적으로 재다운로드됨, 안전 네거티브를 수정하고 생성한 이미지의 메타(네거티브 프롬프트)에 반영됐는지 확인
- [ ] **T7.2 접근성**: VoiceOver 켜고 사이드바·설정 화면 탐색, 키보드만으로 전 화면 조작(Tab/Shift+Tab/Enter/Esc), 언어를 en으로 전환 후 사이드바가 영문으로 바뀌는지 + 미번역 화면에 ko가 남아있어도 깨지지 않는지
- [x] **T7.4 배포 파이프라인 자체**: `v0.1.0` 태그 push → release 워크플로 실행 → draft release에 `.dmg`(`LocalBrush_0.1.0_aarch64.dmg`)·`.app.tar.gz` 첨부 확인 → publish 완료([Release](https://github.com/claire-yunsumin/makesource/releases/tag/v0.1.0)) → tap([homebrew-localbrush](https://github.com/claire-yunsumin/homebrew-localbrush))의 sha256을 실제 dmg 해시로 갱신, `brew info --cask claire-yunsumin/localbrush/localbrush`로 버전·arm64 요구사항 반영 확인 — 이 인텔 기기에서 CLI로 확인 가능한 부분까지 완료
  - [ ] **집 맥에서 남은 확인**: 실제로 다운로드한 dmg를 열어 무서명 상태(우클릭 → 열기 필요)로 설치되는지, README 안내와 실제 동작이 일치하는지, Apple Silicon에서 `brew install --cask claire-yunsumin/localbrush/localbrush`로 실제 설치까지 되는지(이 기기는 arm64 요구사항 때문에 설치 자체를 실행할 수 없음)

## 8. 8GB 메모리 폴백 시나리오 (별도 8GB 기기 또는 메모리 제한 환경)

- [ ] 8GB 기기에서 최초 설치 시 RAM 감지로 "라이트" 프로파일이 기본 추천되는지
- [ ] 표준(SDXL) 프로파일로 무리하게 생성 시도 → OOM 발생 → ① 해상도 하향 → ② SD1.5 폴백 순으로 자동 재시도되는지, 각 단계마다 사용자 고지 문구("메모리가 부족해 …")가 뜨는지
- [ ] 에센스 스타일 생성 중 SD1.5 폴백이 걸리면 IP-Adapter가 해제되고 그 사실이 고지되는지 (T4.3)
- [ ] 폴백 후에도 최종적으로 이미지가 생성·저장되는지 (완전 실패가 아니라 저하된 품질로라도 성공)
