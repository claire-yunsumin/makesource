# 로컬 AI 그래픽 생성기 (가칭: LocalBrush) — 프로젝트 개요 및 실행계획

작성일: 2026-07-05 | 버전: v0.1 (Draft)

---

## 1. 프로젝트 배경

토스의 사내 AI 그래픽 생성기 '토스트(tosst)'는 자사 그래픽 2,000여 개를 Stable Diffusion에 학습시키고, 비전문가도 "이미지 타입 선택 + 개체 키워드 입력"만으로 브랜드 톤의 그래픽을 뽑을 수 있도록 프롬프트를 뒷단에서 조립해주는 구조다.

본 프로젝트는 이 개념을 **개인/팀이 소유한 맥에 설치해서 쓰는 로컬 데스크톱 앱**으로 구현한다. 핵심 차별점은 다음과 같다.

- **완전 로컬**: 이미지 업로드(학습 데이터), 생성, 다운로드가 모두 사용자 기기 안에서 이루어짐 → 브랜드 자산 유출 위험 없음, API 비용 없음
- **설치형**: .dmg 배포, 별도 서버·계정 불필요
- **프리셋 기반 UX**: 프롬프트 지식 없이 "타입 + 키워드"만으로 생성

## 2. 토스트(토스)·에센스 자산화(배민) 사례에서 가져올 핵심 설계 원칙

| 참고 사례의 접근 | 본 프로젝트 적용 |
|---|---|
| 브랜드 그래픽 톤을 언어로 정의 후 학습 | 사용자가 자기 브랜드 이미지를 업로드해 LoRA 학습 (또는 사전 학습된 스타일 LoRA 사용) |
| 저작권 자유로운 자체 리소스 2,000개 학습 | 사용자 소유 이미지만 학습 대상으로 허용 (약관·가이드 명시) |
| AI가 잘 만든 결과물을 재학습시켜 품질 향상 | "즐겨찾기한 생성 결과를 재학습 데이터로 추가" 기능 (Phase 3) |
| 필수 입력 2가지: 이미지 타입 + 개체 | 동일. 타입별 prefix/suffix 프롬프트를 JSON 프리셋으로 관리 |
| 프리셋 프롬프트를 디자이너가 연구·관리 (토스트) | 프리셋 편집기 제공 → 고급 사용자가 직접 튜닝 가능 |
| **에센스를 뽑아 자산화하면 신입도 시니어 수준의 일관된 결과물 (배민/우아한형제들)** | 참조 이미지에서 스타일 에센스를 자동 추출하는 마법사, 학습 없이 즉시 사용 (§3.6) |
| **자산화의 3전제: 성공 기준·테스트 환경(버전 관리)·괜찮은 첫 프롬프트 (배민)** | 프리셋 버전 관리 + 성공 기준 메모 + 시드 고정 A/B 비교 생성 |

## 3. 기술 접근 방법

### 3.1 이미지 생성 모델

| 후보 | 장점 | 단점 | 판단 |
|---|---|---|---|
| **SDXL 1.0 + LoRA** | 생태계 성숙, LoRA 학습 자료 풍부, 라이선스(OpenRAIL) 상용 가능 | FLUX 대비 최신 품질 열세 | **MVP 기본 모델** |
| FLUX.1 [schnell] | 품질 우수, Apache 2.0 | LoRA 학습 도구 상대적으로 무거움, 메모리 요구 큼 | Phase 3 옵션 |
| SD 1.5 | 가볍고 빠름, 저사양 대응 | 해상도·품질 한계 | 8~16GB RAM 폴백 |

※ 모델·라이선스 상황은 빠르게 변하므로 착수 시점에 재검증 필요.

### 3.2 Apple Silicon 로컬 추론

| 방식 | 설명 | 판단 |
|---|---|---|
| **Core ML (apple/ml-stable-diffusion)** | 애플 공식, Neural Engine 활용, 네이티브 통합 최적 | 네이티브 앱이면 1순위 |
| MLX | 애플의 ML 프레임워크, LoRA 학습까지 로컬 가능 | 로컬 학습 기능(Phase 2)의 핵심 후보 |
| Python diffusers + MPS | 유연성 최고, 생태계 그대로 사용 | 파이프라인 직접 구현 필요 | GPL 회피 필요 시 대안 |
| **ComfyUI 헤드리스 내장** | 생성·IP-Adapter·업스케일 파이프라인 기구현 → 개발 공수 최소 | GPL-3.0(소스 공개 의무 고려), 번들 용량 | **무료 직접 구축 시 1순위** (localhost API로 연동) |

### 3.3 앱 아키텍처 (2안 비교)

**A안 — SwiftUI 네이티브 + Core ML** (권장)
- 프론트: SwiftUI / 추론: Core ML 변환 모델 / 학습: MLX 서브프로세스
- 장점: 성능·전력효율·배포(.dmg, 공증) 최적, 앱 용량 작음
- 단점: 최신 모델 대응 속도가 Python 생태계보다 느림

**B안 — Tauri(웹 프론트) + Python 사이드카(diffusers/kohya)**
- 장점: 개발 속도 빠름, 최신 모델·학습 파이프라인 즉시 수용
- 단점: 번들 용량 큼(Python 런타임 포함 수 GB), 네이티브 대비 성능 손해

→ **결론: MVP는 B안으로 빠르게 검증 후, 제품화 단계에서 A안 전환 또는 하이브리드**(UI는 Tauri 유지, 추론만 Core ML 교체) 검토. 개발 리소스가 Swift에 강하면 처음부터 A안.

### 3.4 브랜드 스타일 학습 (LoRA)

- 입력: 사용자 이미지 20~200장 + 캡션(자동 캡셔닝: 로컬 VLM 또는 규칙 기반 태깅 → 사용자 수정)
- 학습: MLX 기반 LoRA 트레이너 또는 kohya_ss 파이프라인 내장
- 예상 소요: M3 Pro/Max 기준 SDXL LoRA 수 시간 수준(장수·스텝에 따라 큼) → 진행률 표시 + 백그라운드 학습 필수
- 대안 경로: 로컬 학습이 부담스러운 사용자를 위해 "학습 없이 사전 제공 스타일 LoRA + IP-Adapter(참조 이미지 스타일 반영)" 경로를 MVP에 포함

### 3.5 프리셋 프롬프트 시스템

토스트의 "동화같은 → `creating dreamlike atmosphere, soft pastel colors`" 방식처럼:

```json
{
  "preset_id": "storybook",
  "label": "동화같은",
  "prefix": "cinematic illustration of",
  "suffix": "creating dreamlike atmosphere, soft pastel colors",
  "negative": "text, watermark, photo-realistic",
  "params": { "steps": 28, "cfg": 6.5, "size": "1024x1024" }
}
```

- 프리셋은 로컬 JSON으로 저장, 앱 내 편집기로 추가/수정/공유(내보내기·가져오기) 가능
- **프리셋 = 팀 자산**: 각 프리셋에 버전과 "성공 기준 메모"(어떤 결과가 나오면 성공인지)를 기록하고, 동일 시드로 버전 간 비교 생성(A/B)을 지원해 프리셋 개선을 반복 가능한 프로세스로 만든다

### 3.6 스타일 에센스 추출 (학습 없는 브랜드 톤 재현) — 배민 사례 반영

우아한형제들 사례의 핵심은 파인튜닝 없이도 **스타일의 '에센스'를 언어(프롬프트)로 추출해 자산화**하면 신입도 시니어 수준의 일관된 결과물을 낼 수 있다는 것이다. 본 프로젝트에서는 이를 LoRA 학습과 병행하는 제1경로로 채택한다.

- **에센스 마법사**: 사용자가 참조 이미지 3~10장을 드롭하면 로컬 VLM(경량 비전 언어 모델)이 스타일을 분석해 에센스 프롬프트(형태·텍스처·컬러·구도 서술)를 자동 생성 → 사용자가 다듬어 프리셋으로 저장
- IP-Adapter(참조 이미지 스타일 주입)와 결합 시 학습 없이도 상당한 톤 일관성 확보 가능
- **두 경로의 역할 분담**: 에센스 프리셋 = 설치 직후 즉시 사용·빠른 반복(수 분), LoRA 학습 = 톤 재현 정밀도가 필요할 때(수 시간). 온보딩은 에센스 → 만족도 부족 시 LoRA 학습 제안 순서로 설계

## 4. 최소 시스템 요구사항 (초안)

- macOS 14 Sonoma 이상, Apple Silicon (M1 이상)
- RAM 16GB 권장 (8GB는 SD 1.5 폴백 모드)
- 디스크 여유 공간 15GB 이상 (모델 파일 포함)

## 5. 단계별 실행 계획

| 단계 | 기간(예상) | 내용 | 산출물 |
|---|---|---|---|
| **Phase 0. 검증(PoC)** | 2~4주 | SDXL LoRA를 샘플 브랜드 이미지로 학습 → M 시리즈 맥에서 추론 속도·품질 측정 | 기술검증 보고서, Go/No-Go |
| **Phase 1. MVP** | 8~10주 | 생성(프리셋+키워드) / 갤러리 / PNG 다운로드 / 사전 스타일 LoRA 3종 번들 / .dmg 배포 | 내부 베타 앱 |
| **Phase 2. 학습 기능** | 6~8주 | 이미지 업로드 → 자동 캡셔닝 → 로컬 LoRA 학습 → 내 스타일로 생성 | v1.0 정식 릴리스 |
| **Phase 3. 고도화** | 지속 | 배치 생성, 업스케일(Real-ESRGAN), 즐겨찾기 재학습, FLUX 지원, 프리셋 마켓 | v1.x |

## 6. 주요 리스크와 대응

| 리스크 | 영향 | 대응 |
|---|---|---|
| 로컬 LoRA 학습이 너무 느림 (수 시간+) | 핵심 UX 훼손 | 백그라운드 학습 + 알림 / 학습 없는 에센스·IP-Adapter 경로 병행 / SD 1.5 LoRA로 다운시프트 / (선택) Colab·Kaggle 무료 GPU — 단, 이미지가 기기를 떠나므로 로컬 원칙과 트레이드오프 |
| 8~16GB 기기에서 OOM | 사용자층 축소 | 해상도·모델 자동 다운그레이드, 요구사항 명확 고지 |
| 모델 라이선스 변경 | 상용화 차질 | OpenRAIL/Apache 계열만 채택, 착수 시 법무 검토 |
| 학습 데이터 저작권 (사용자가 타인 이미지 학습) | 법적 리스크 | 약관에 사용자 책임 명시, 온보딩에서 고지 |
| 앱 번들 용량 (모델 포함 10GB+) | 배포·설치 마찰 | 최초 실행 시 모델 다운로드 방식(로컬 저장), 앱 본체는 경량 유지 |

## 7. 비용 및 수행 체계 — 전액 무료 구축

- **금전 비용 0원 원칙**: 외주 없이 직접 구축. 모델 가중치(Hugging Face), 추론·학습 도구(ComfyUI, kohya, diffusers), 앱 프레임워크(Tauri/SwiftUI), 배포(GitHub Releases)까지 전부 무료 오픈소스 사용 — 상세 BOM은 `01_제로코스트_구축계획.md`
- 유일한 선택적 지출은 Apple 개발자 계정(연 $99, 공증용)이며 우클릭 실행 안내로 생략 가능
- 수행 인력: 1인 개발 가능 (풀타임 약 2~3개월 / 사이드 프로젝트 4~6개월). 프리셋 프롬프트 연구는 토스트 사례처럼 디자인 감각이 필요하므로, 가능하면 디자이너 1인의 파트타임 협업 권장 (무보수 협업 또는 본인이 겸임)
