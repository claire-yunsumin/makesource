# 결정기록 (Decision Log)

버전: v0.1 | 작성일: 2026-07-05 | 상태: 운영 중

프로젝트의 주요 기술·운영 결정을 기록한다. 초기 결정(D-001~D-006)은 소급 기록.

**기록 규칙**
- 새 결정은 맨 아래에 D-번호를 이어서 추가하고, 번복 시 원래 항목을 지우지 말고 상태를 `대체됨 → D-xxx`로 바꾼다
- "결정"은 되돌리기 비용이 큰 선택(스택, 라이선스, 계약, 운영 정책)만 기록한다. 구현 세부는 PR 설명으로 충분

## D-001. 앱 셸: Tauri 2 + ComfyUI 별도 프로세스

- **날짜**: 2026-07-05 | **상태**: 확정
- **결정**: SwiftUI+Core ML 대신 Tauri 2(React/TS + Rust)를 앱 셸로, 추론은 ComfyUI를 별도 프로세스로 기동해 HTTP/WS로 통신한다.
- **이유**: 제로코스트 구축 기본안(docs/01). ComfyUI 생태계(워크플로·커스텀 노드)를 그대로 활용하면서 앱 코드는 GPL 전염 없이 유지 가능. 웹 프론트 역량 재사용.
- **영향**: ComfyUI는 저장소에 포함하지 않고 부트스트랩 설치기(T1.1)가 사용자 기기에 설치. PRD §8-1 해소.

## D-002. 저장소 public 전환 + main 브랜치 보호

- **날짜**: 2026-07-05 | **상태**: 확정
- **결정**: 저장소를 public으로 전환하고, main에 `protect-main` 룰셋 적용 (직접 push 금지, PR 필수, CI 통과 후 squash merge만 허용).
- **이유**: GitHub 무료 플랜에서 브랜치 보호 룰셋은 public 저장소에서만 동작. 트렁크 기반 전략(CLAUDE.md)의 "main 항상 그린"을 강제하기 위해 필요.
- **영향**: 공개 저장소가 되었으므로 라이선스 명시가 필요해짐 → D-006.

## D-003. DB: sqlx + SQLite

- **날짜**: 2026-07-05 (T0.3, PR #3) | **상태**: 확정
- **결정**: 로컬 DB는 SQLite, Rust 접근 계층은 rusqlite가 아닌 sqlx(runtime-tokio, migrate) 채택.
- **이유**: 컴파일 타임 쿼리 검증(macros), 내장 마이그레이션, async 런타임(tokio)과의 자연스러운 결합. Tauri command가 async이므로 동기 API인 rusqlite는 spawn_blocking 래핑이 반복적으로 필요.
- **영향**: 스키마 변경은 `migrations/`에 마이그레이션 파일 추가로만 수행. 스키마 정의는 TAD §4가 기준.

## D-004. 앱 데이터 경로: `~/Library/Application Support/LocalBrush/`

- **날짜**: 2026-07-05 (T0.3 fix, PR #4) | **상태**: 확정
- **결정**: 앱 데이터 루트를 TAD §3에 정의된 `LocalBrush/` 단일 폴더로 통일. 히스토리 이미지 경로는 이 루트 기준 상대 경로로 저장.
- **이유**: 초기 구현이 TAD와 다른 경로를 사용해 정합성 수정. 사용자가 저장 위치를 옮겨도 DB 레코드가 깨지지 않아야 함.

## D-005. long-running 작업: jobId + Tauri event 패턴

- **날짜**: 2026-07-05 (T1.1~) | **상태**: 확정
- **결정**: 오래 걸리는 command(부트스트랩, 생성, 학습)는 jobId만 즉시 반환하고 진행 상황은 Tauri event(`bootstrap://`, `gen://`, `train://`)로 push한다. 블로킹 command 금지.
- **이유**: UI 프리즈 방지, 취소·재시작 제어 지점 확보. command 시그니처는 TAD §5가 단일 기준.

## D-006. 라이선스: MIT

- **날짜**: 2026-07-05 | **상태**: 확정
- **결정**: 저장소 코드를 MIT 라이선스로 공개. 저작권자는 Claire Yun.
- **이유**:
  - ComfyUI(GPL-3.0)는 저장소에 포함하지 않고 사용자 기기에 별도 설치된 독립 프로세스와 통신만 하므로(D-001) GPL 전염 없음 — 라이선스 자유 선택 가능
  - PRD §8-4 "향후 유료화 가능성"에 열려 있으려면 허용적(permissive) 라이선스가 적합. 저작권자가 단독이므로 향후 조건 변경도 자유로움
  - 개인 공개 저장소의 사실상 표준. Apache-2.0의 특허 조항은 현 단계에서 과함
- **영향**: 새 의존성은 MIT/Apache/BSD 우선, GPL 추가 금지 원칙(CLAUDE.md 절대 규칙 2) 유지. PRD §8-2 해소.

## D-007. 기본 프리셋 축: 용도(이미지 타입) 기반 6종

- **날짜**: 2026-07-06 | **상태**: 확정
- **결정**: `presets.default.json`의 기본 프리셋을 그림체 기반 6종(플랫 벡터/수채화/라인 아트 등)에서 **용도 기반 6종**(기본/동화같은/인포그래픽/아이콘/3D 오브젝트/배너 — PRD F-2.1 예시 그대로)으로 교체한다.
- **이유**: 제품 설계상 두 축은 직교해야 한다 — **스타일 = 브랜드 톤·그림체**(에센스/LoRA가 담당, PRD Flow A의 "플랫 일러스트" 예시), **이미지 타입 = 용도**(인포그래픽/아이콘/배너). 초기 구현이 그림체를 이미지 타입 자리에 넣어 두 축이 겹쳤고, 사용자가 브랜드 스타일을 만들면 수채화·플랫벡터류 프리셋과 톤이 충돌하는 UX 문제가 생긴다. 토스트 원형("이미지 타입 + 개체")과도 용도 축이 부합.
- **영향**: 구 그림체 5종의 프롬프트 값은 폐기하지 않고 07 문서 부록 A에 "번들 스타일 후보"로 보존 (M4 에센스/번들 LoRA에서 재사용). `storybook`은 양쪽 목록에 공통이라 id 유지, 구 `3d-render` 값은 `3d-object`로 승계. 07 연구 문서는 새 6종 기준 v0.2로 재작성.
