# UI / 디자인 스펙
## LocalBrush — 화면 구조, 디자인 토큰, 컴포넌트

버전: v0.1 | 상위 문서: 02_PRD.md, 03_기술설계서_TAD.md

---

## 1. 디자인 원칙

1. **입력은 2개면 충분** — 토스트 원칙. 기본 화면에는 "스타일·타입 선택 + 키워드"만 노출, 나머지는 고급 패널로 접기
2. **기다림을 설계** — 생성(수십 초)·학습(수 시간)·설치(수 분)가 잦은 앱. 모든 대기에는 진행률 + 예상시간 + 취소를 제공
3. **로컬 신뢰 시각화** — "이 작업은 기기 안에서만 처리됩니다" 상태 표시를 상시 노출 (오프라인 아이콘)
4. 다크 모드 우선 설계(그래픽 확인에 유리), 라이트 모드 지원

## 4. 화면별 스펙

### 4.1 생성 (메인, `/generate`)

```
┌ 좌측 패널 (320px) ─────────┐ ┌ 우측 결과 영역 ──────────────┐
│ [스타일 선택]  칩/드롭다운   │ │  결과 그리드 2×2              │
│   (없음 | 에센스 | LoRA)   │ │  각 셀: 이미지, hover 시       │
│ [이미지 타입]  카드 6개      │ │   [♥][⬇ PNG][시드 재생성][ⓘ] │
│   썸네일 + 라벨            │ │  생성 중: 셀별 진행률 오버레이   │
│ [키워드 입력]  textarea     │ │  빈 상태: 온보딩 안내 일러스트   │
│   placeholder "예: 통나무집"│ │                              │
│ [장수 1-4 | 크기 3옵션]     │ │                              │
│ ▸ 고급 (프롬프트 미리보기,   │ │                              │
│    시드, steps, cfg 슬라이더)│ │                              │
│ [✦ 생성하기]  primary, 대형 │ │                              │
└────────────────────────────┘ └──────────────────────────────┘
```

상태: idle / generating(버튼→취소로 변경) / error(토스트+패널 상단 배너)
단축키: ⌘↵ 생성, Esc 취소

### 4.2 갤러리 (`/gallery`)

- 상단: 검색 인풋(키워드), 필터 칩(스타일/프리셋/♥), 정렬(최신)
- 본문: masonry 그리드, 무한 스크롤(cursor 기반)
- 항목 클릭 → 상세 모달: 원본 미리보기, 전체 메타(프롬프트/시드/파라미터 복사 버튼), [같은 설정으로 다시 생성], [학습 데이터에 추가(P2)], 내보내기(PNG/JPG/WebP, 투명 배경 토글)
- 드래그 아웃 → Finder로 파일 내보내기 (tauri drag)

### 4.3 스타일 (`/styles`)

- 스타일 카드 그리드: 썸네일, 이름, kind 배지(`에센스`/`LoRA`), [생성에 사용] 버튼
- [+ 새 스타일] → 분기 다이얼로그:
  - **빠른 스타일 (에센스)** — "몇 분, 학습 불필요" 배지
  - **정밀 학습 (LoRA)** — "수 시간, 고품질" 배지
- **에센스 마법사 (3단계 스텝퍼)**: ① 이미지 3~10장 드롭존 → ② 분석 중(진행률) → 생성된 에센스 프롬프트 편집 textarea + IP-Adapter 강도 슬라이더 → ③ 샘플 4장 생성해 확인 → 저장
- **학습 마법사 (4단계)**: ① 이미지 20장+ 드롭(부족 시 경고 배너) → ② 캡션 테이블(썸네일|캡션 인라인 편집|일괄 찾아바꾸기) → ③ 프로파일 선택(빠름/표준/고품질, 각각 예상시간 표기) → ④ 학습 대시보드(진행률 바, ETA, epoch 샘플 이미지 스트립, 취소). 백그라운드로 보내고 사이드바 큐에서 추적 가능

### 4.4 프리셋 (`/presets`)

- 좌: 프리셋 리스트(버전 배지 vN) / 우: 편집 폼(prefix, suffix, negative, params, 성공 기준 메모)
- 상단 액션: [버전 저장] [이전 버전 보기/복원] [A/B 비교] [내보내기/가져오기]
- A/B 비교 뷰: 동일 시드·키워드로 좌우 2열 생성 결과 나란히 표시

### 4.5 설정 (`/settings`)

- 모델 관리: 설치된 모델 목록·용량, 저장 위치 변경, 캐시 정리
- 생성 기본값, 언어(ko/en), 안전 네거티브 편집
- 정보: 오픈소스 라이선스 목록(BOM), 버전, 업데이트 확인

### 4.6 최초 실행 (부트스트랩 풀스크린)

- 단계 리스트(체크 진행): 환경 준비 → 엔진 설치 → 모델 다운로드(프로파일 선택: 표준 ~10GB / 라이트 ~4GB, RAM 감지로 추천 배지) → 준비 완료
- 실패 시: 에러 요약 + [로그 열기] + [재시도]

## 5. 공용 컴포넌트 목록 (src/components/)

## 6. 문구 톤 & 상태 메시지

- 존댓말, 짧게. 기술 용어 최소화 (예: "OOM" → "메모리가 부족해 크기를 낮춰 다시 시도했어요")
- 대기 문구는 구체적으로: "이미지를 만드는 중이에요 · 약 20초 남음"
- 실패는 원인+다음 행동: "엔진이 응답하지 않아요. 다시 시작해볼까요? [엔진 재시작]"

## 7. 접근성

- 모든 인터랙티브 요소 키보드 접근, 포커스 링 유지
- 이미지 셀에 aria-label(키워드+프리셋), 진행률에 aria-live
- 색 대비 WCAG AA 이상 (토큰이 이를 만족하도록 검증)
