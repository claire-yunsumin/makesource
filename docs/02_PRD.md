# 제품요구사항정의서 (PRD)
## LocalBrush — 로컬 AI 브랜드 그래픽 생성기 (macOS)

버전: v0.1 Draft | 작성일: 2026-07-05 | 상태: 검토 중

---

## 1. 제품 개요

### 1.1 한 줄 정의
내 브랜드 이미지를 내 맥에서 학습시키고, 키워드만 입력하면 브랜드 톤의 그래픽을 생성·다운로드하는 설치형 macOS 앱.

### 1.2 문제 정의
- 스타트업/소규모 팀은 그래픽 디자이너 리소스가 부족해 아이콘·콘텐츠 이미지 수요를 감당하지 못함 (토스도 디자이너 5명이 100명+ 협업 수요를 감당하다 토스트를 만듦)
- 기존 생성형 AI(Midjourney 등)는 ① 브랜드 톤 일관성이 없고 ② 프롬프트 작성 지식이 필요하며 ③ 브랜드 자산을 외부 서버에 올려야 함
- AI 활용 노하우가 개인의 프롬프트 실력에 갇혀 있으면 팀 차원의 일관된 결과물이 나오지 않음 — 우아한형제들 사례처럼 스타일 에센스를 "자산"으로 만들어 누가 써도 같은 톤이 나오게 해야 함
- 클라우드 생성 서비스는 반복 사용 시 API 비용이 누적됨

### 1.3 솔루션
1. **스타일 에센스 추출 (학습 불필요, 즉시 사용)**: 참조 이미지 몇 장에서 로컬 VLM이 스타일 에센스 프롬프트를 자동 추출 → IP-Adapter와 결합해 브랜드 톤 재현. 배민(우아한형제들)의 "에센스 자산화" 접근
2. **로컬 스타일 학습 (정밀 재현)**: 사용자 이미지 20장 이상으로 LoRA를 기기 내에서 학습. 토스트의 파인튜닝 접근
3. **프리셋 UX**: "이미지 타입 선택 + 개체 키워드 입력" 2단계만으로 생성. 프롬프트 엔지니어링은 프리셋이 뒷단에서 처리하며, 프리셋은 버전 관리되는 팀 자산으로 취급
4. **완전 로컬**: 업로드·학습·생성·저장 전 과정이 기기 안에서 종결

### 1.4 타깃 사용자 (Persona)

| 페르소나 | 상황 | 핵심 니즈 |
|---|---|---|
| **P1. 스타트업 마케터 (주 타깃)** | 디자이너 없이 블로그 썸네일·SNS 이미지 주 10장 필요 | 프롬프트 몰라도 브랜드 톤 이미지 즉시 생성 |
| P2. 1인 브랜드 디자이너 | 반복적 그래픽 요청에 시간 소진 | 자기 그림체를 학습시켜 초안 자동화, 자산 보안 |
| P3. 사내 콘텐츠 팀 | 브랜드 가이드 준수 필수, 보안 정책상 외부 업로드 금지 | 오프라인 동작, 팀 프리셋 공유 |

| 생성 이미지 다운로드율 (생성 대비) | 30% 이상 (품질 대리 지표) |

## 2. 범위

### 2.1 포함 (v1.0)
생성, 갤러리, 다운로드, 프리셋 시스템, 번들 스타일 LoRA, 로컬 LoRA 학습

### 2.2 제외 (Non-goals)
- Windows/Linux 지원, 모바일 앱
- 클라우드 동기화·협업 기능 (프리셋 파일 내보내기로 대체)
- 이미지 편집기(레이어, 브러시 등) — 생성기이지 에디터가 아님
- 실사 인물 사진 생성 특화 (브랜드 그래픽/일러스트 중심)

## 3. 사용자 시나리오 및 기능 요구사항

### 3.1 핵심 플로우

**Flow A. 온보딩 → 첫 생성 (P1, 학습 없이)**
1. .dmg 설치 → 첫 실행 → 기기 사양 확인 및 권장 모델 자동 선택
2. 기본 모델 + 번들 스타일 LoRA 다운로드 (진행률 표시)
3. 스타일 선택(예: "플랫 일러스트") → 이미지 타입 선택(예: "인포그래픽") → 키워드 입력("통나무집") → 생성
4. 4장 그리드 미리보기 → 마음에 드는 것 PNG 다운로드

**Flow B. 스타일 에센스 만들기 (P1·P3, 학습 없이 브랜드 톤)**
1. "새 스타일 만들기" → "빠른 스타일(에센스)" 선택 → 참조 이미지 3~10장 드롭
2. 로컬 VLM이 스타일 분석 → 에센스 프롬프트 자동 생성(형태·텍스처·컬러·구도 서술)
3. 사용자가 문구 다듬기 → 샘플 생성으로 확인 → 스타일 프리셋으로 저장 (수 분 내 완료)
4. 이후 Flow A와 동일하게 생성. 결과가 아쉬우면 "정밀 학습(LoRA)으로 업그레이드" 제안

**Flow C. 내 브랜드 스타일 정밀 학습 (P2)**
1. "새 스타일 만들기" → 이미지 20~200장 드래그앤드롭
2. 자동 캡셔닝 결과 확인·수정 → 학습 시작
3. 백그라운드 학습 (진행률·예상시간, 완료 시 macOS 알림)
4. 완료 후 스타일 목록에 추가 → Flow A와 동일하게 생성

## 5. 콘텐츠 안전 및 법적 고려

- 기본 네거티브 프롬프트 및 안전 필터로 부적절 콘텐츠 생성 억제 (로컬 앱 특성상 완전 차단은 불가함을 인지)
- 온보딩·약관에 명시: 학습 이미지는 사용자가 권리를 보유한 것만 사용, 생성물의 상업적 이용 책임은 사용자에게 있음
- 사용 모델·라이브러리 라이선스 고지 화면 (BOM)

## 8. 미결정 사항 (Open Questions)

1. 앱 셸: Tauri+ComfyUI(무료 구축 기본안) vs SwiftUI+Core ML — Step 0/1 검증 결과와 GPL 소스 공개 여부 판단으로 결정
2. 오픈소스 공개 범위: ComfyUI(GPL) 내장 시 앱 소스 공개가 사실상 전제 — 전체 공개(권장) vs diffusers 경로로 전환해 비공개 유지
3. 한→영 변환: Argos Translate vs Ollama 경량 LLM vs 도메인 용어 사전 — 품질·용량 비교 필요
4. 수익화: 당분간 무료 배포 전제. 향후 유료화 시에도 현재 스택은 전부 상용 이용 가능 라이선스로 구성해 전환 비용 없음
5. 배포 채널: GitHub Releases + Homebrew cask (무료). Mac App Store는 개발자 계정 비용·샌드박스 제약으로 보류
