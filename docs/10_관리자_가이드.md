# LocalBrush 관리자 가이드

LocalBrush는 서버나 중앙 관리 콘솔이 없는 **완전 로컬 데스크톱 앱**입니다. 각 팀원의 Mac에 각자 설치되고, 데이터도 각 기기 안에만 저장됩니다. 이 문서는 "여러 명이 쓰는 환경을 셋업·유지보수하는 사람"(IT 담당자, 팀 리드, 또는 혼자 여러 기기를 관리하는 사람)을 위한 것입니다. 일반 사용법은 [docs/09_사용자_가이드.md](09_사용자_가이드.md)를 먼저 참고하세요.

---

## 1. 배포하기

### 개별 설치 (기본)

각 사용자가 [Releases](https://github.com/claire-yunsumin/makesource/releases)에서 `.dmg`를 받아 직접 설치합니다. 무서명 배포라 "확인되지 않은 개발자" 경고가 뜨는데, 사용자 가이드의 "우클릭 → 열기" 안내를 팀에 미리 공지해두면 문의를 줄일 수 있습니다.

### Homebrew로 여러 대에 배포

사내에 Mac이 여러 대라면 Homebrew가 더 간단합니다.

```bash
brew install --cask claire-yunsumin/localbrush/localbrush   # 설치
brew upgrade --cask localbrush                               # 업데이트
brew uninstall --cask localbrush                              # 제거
```

캐스크 정의는 별도 tap 저장소 [homebrew-localbrush](https://github.com/claire-yunsumin/homebrew-localbrush)에 있습니다. **주의**: 이 tap의 버전·체크섬은 릴리스마다 수동으로 갱신됩니다(자동화 없음) — 새 릴리스 직후에는 `brew upgrade`로 최신 버전이 안 잡힐 수 있으니, 급하면 dmg 직접 재설치를 안내하세요.

### 버전 관리

- 릴리스는 `vX.Y.Z` 태그 push → GitHub Actions가 dmg를 빌드해 **Release 초안(draft)**으로 올립니다. 태그를 push한다고 바로 공개되지 않으니, [Releases](https://github.com/claire-yunsumin/makesource/releases) 페이지에서 draft를 확인하고 수동으로 "Publish"해야 사용자에게 보입니다.
- 지금은 앱 내 자동 업데이트 기능이 없습니다(F-5.4, 후속 예정). 새 버전이 나오면 팀에 직접 공지하거나 Homebrew(`brew upgrade`)를 안내하세요.

## 2. 데이터가 어디에 저장되는지

모든 사용자 데이터는 각 Mac의 아래 폴더 안에만 있습니다 (외부로 전송되지 않음):

```
~/Library/Application Support/LocalBrush/
├── models/           # 다운로드된 AI 모델 (checkpoints, loras, ipadapter 등) — 용량 대부분 차지
├── runtime/          # ComfyUI/kohya/venv 실행 환경 (재설치로 복구 가능, 백업 불필요)
├── outputs/YYYY-MM/  # 생성된 이미지 원본
├── training/         # 학습 중/완료된 데이터셋
├── styles/           # 에센스 스타일의 참조 이미지
├── presets.json      # 프리셋 (버전 이력 포함)
├── styles.json       # 등록된 스타일 목록
├── settings.json     # 앱 설정 (안전 네거티브, 언어 등)
├── app.db            # 생성 히스토리 DB
└── logs/             # bootstrap.log, engine.log 등
```

**백업이 필요한 것**: `presets.json`, `styles.json`, `settings.json`, `app.db`, `outputs/`, `styles/`(참조 이미지), `training/`(원본 데이터셋). **백업 불필요**: `models/`, `runtime/` — 용량이 크지만 재설치 시 자동으로 다시 받아집니다.

### 다른 Mac으로 옮기기

위 "백업이 필요한 것" 목록을 새 기기의 같은 경로에 복사하면 프리셋·스타일·생성 히스토리가 그대로 이어집니다. `models/`·`runtime/`은 새 기기에서 앱이 최초 실행 부트스트랩으로 알아서 다시 받으므로 옮길 필요 없습니다.

## 3. 팀 자산 공유

PRD상 프리셋은 "팀 자산"으로 취급합니다. 현재 앱에서 공식 지원하는 공유 방법은 **프리셋뿐**입니다.

- **프리셋 공유**: 프리셋 화면 → [내보내기]로 파일 저장 → 팀원에게 전달(사내 드라이브, 메신저 등 — 프리셋 자체엔 이미지가 없어 용량이 작음) → 받는 사람이 [가져오기]로 불러오기. 버전이 다른 파일을 가져오면 실패하고 아무것도 바뀌지 않으니 안전합니다.
- **스타일(에센스/LoRA) 공유**: 아직 내보내기/가져오기 UI가 없습니다. 공유하려면 관리자가 직접 아래 폴더를 복사해 전달해야 합니다.
  - 에센스 스타일: `styles/{style_id}/`(참조 이미지) + `styles.json`의 해당 항목
  - LoRA 스타일: `models/loras/`의 `.safetensors` 파일 + `styles.json`의 해당 항목
  - `styles.json`은 JSON 배열이므로, 항목을 추가할 때는 다른 스타일을 건드리지 않게 조심스럽게 병합해야 합니다(앱을 완전히 종료한 상태에서 편집 권장).

## 4. 저장 공간·모델 관리

설정 화면(앱 안)에서 대부분 처리됩니다 — 사용자에게 직접 안내하거나 관리자가 원격으로 확인할 때 참고:

- **모델 관리**: 카테고리별(체크포인트/LoRA/IP-Adapter 등) 설치 목록과 용량 표시
- **캐시 정리**: `models/hf`(에센스 분석용 캐시)를 비웁니다. 다음에 에센스 기능을 쓰면 자동으로 다시 받으므로 언제 지워도 안전합니다
- 표준 프로파일은 약 10GB, 라이트 프로파일은 약 4GB — 회의실 공용 Mac처럼 저장 공간이 빠듯한 기기에는 라이트 프로파일 설치를 권장하세요

터미널에서 직접 확인하려면:

```bash
du -sh ~/Library/Application\ Support/LocalBrush/models/*
```

## 5. 문제 해결

| 증상 | 확인할 것 |
|---|---|
| 설치가 멈추거나 실패 | 앱의 [로그 열기] 또는 직접 `~/Library/Application Support/LocalBrush/logs/bootstrap.log` 확인. 대개 네트워크 문제 — 재시도로 해결됨(이어받기 지원) |
| 생성이 안 되거나 "엔진이 응답하지 않아요" | `logs/engine.log` 확인. 앱을 완전히 종료(⌘Q) 후 재실행하면 대부분 해결. 활동 모니터에서 python(ComfyUI) 프로세스가 남아있으면 강제 종료 후 재실행 |
| 메모리 부족(OOM)이 자주 발생 | 8~16GB 기기는 예상된 동작 — 앱이 자동으로 해상도를 낮추거나 라이트 모델로 전환합니다. 자주 발생하면 라이트 프로파일 재설치를 권장 |
| 설치를 완전히 초기화하고 싶음 | 앱 종료 후 `~/Library/Application Support/LocalBrush/` 폴더 전체 삭제 → 앱 재실행 시 최초 설치부터 다시 시작. **주의: 위 2장의 "백업이 필요한 것" 항목이 사라지니 먼저 백업할 것** |
| 앱 완전 삭제(제거) | Applications 폴더에서 앱 삭제 + `~/Library/Application Support/LocalBrush/` 삭제. Homebrew로 설치했다면 `brew uninstall --cask localbrush --zap`(zap이 위 데이터 폴더까지 정리) |

## 6. 보안·규정 준수 체크리스트 (사내 콘텐츠 팀 대상)

PRD의 P3 페르소나(보안 정책상 외부 업로드 금지)를 위한 확인 항목:

- [ ] 네트워크는 Hugging Face(모델 다운로드)와 GitHub(업데이트 확인)로만 나갑니다. 사내 방화벽 정책과 맞는지 확인 — 필요하면 `huggingface.co`, `github.com`, `githubusercontent.com`을 허용 목록에 추가
- [ ] 이미지·프롬프트가 외부로 전송되지 않는다는 것은 CLAUDE.md의 "절대 규칙"으로 코드베이스 차원에서 강제됩니다(크래시 리포터·텔레메트리 코드 자체가 없음)
- [ ] 학습에 사용하는 이미지는 팀이 저작권을 보유한 것만 쓰도록 안내하세요 — 생성물의 상업적 이용 책임은 사용자(팀)에게 있습니다
- [ ] 앱과 대부분의 구성 요소는 상업적 이용 가능 라이선스지만, 기반 모델(Stable Diffusion 등)은 별도 라이선스 조건이 있습니다. 설정 화면 "오픈소스 라이선스" 또는 [resources/licenses.json](../src-tauri/resources/licenses.json)에서 전체 목록 확인 가능
